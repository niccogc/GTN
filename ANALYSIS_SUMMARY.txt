================================================================================
                              KEY FINDINGS
================================================================================

1. MODEL INSTANTIATION & CONFIGURATION
   ✓ Uses Hydra for configuration management
   ✓ 10 model types available (MPO2, LMPO2, MMPO2, CPDA, TypeI variants, TNML)
   ✓ Models instantiated via registry lookup: NTN_MODELS[cfg.model.name]
   ✓ Parameters built dynamically based on dataset dimensions
   ✓ Special handling for TypeI (variable sites) and TNML (feature encoding)

2. INPUT PREPARATION
   ✓ Standard models: append bias term to features
   ✓ TNML models: encode features (polynomial or Fourier) into higher dimensions
   ✓ NTN: uses quimb Inputs class with pre-computed batches
   ✓ GTN: uses PyTorch DataLoaders with standard tensors
   ✓ Classification: targets converted to one-hot encoding
   ✓ Regression: targets kept as continuous values

3. TRAINING/TESTING
   ✓ Two trainer types: NTN (Newton-based) and GTN (gradient-based)
   ✓ NTN: 20 epochs, ridge regularization with decay schedule
   ✓ GTN: 1000 epochs, standard optimizers (Adam, AdamW, SGD)
   ✓ Both support early stopping with patience and min_delta
   ✓ Model selection based on validation quality (with optional train tiebreaker)
   ✓ Error handling for singular matrices and OOM errors

4. AVAILABLE DATASETS
   ✓ 21 datasets total (11 regression, 10 classification)
   ✓ NO small regression datasets exist!
   ✓ Smallest regression: concrete (165 samples) - Medium
   ✓ Small classification datasets: iris, hearth, breast, wine, car_evaluation
   ✓ Fixed 70/15/15 train/val/test split (seed=42)
   ✓ Standardization applied to numeric features only

5. MODEL TYPE SPECIFICATION
   ✓ Command line: python run.py model=lmpo2 dataset=abalone trainer=gtn
   ✓ Config files: create conf/experiment/my_experiment.yaml
   ✓ Parameter overrides: python run.py model.bond_dim=8 trainer.lr=0.01
   ✓ Grid search: python run.py --multirun model.bond_dim=4,6,8
   ✓ All parameters can be overridden from command line

================================================================================
                         EXECUTION FLOW SUMMARY
================================================================================

INITIALIZATION
  └─ Hydra loads config from conf/ directory
     └─ Merges defaults: model=mpo2, dataset=iris, trainer=ntn
        └─ Sets random seeds for reproducibility

DATASET LOADING
  └─ load_dataset() fetches from UCI ML Repository
     └─ One-hot encodes categorical features (with cap=50)
        └─ Fixed 70/15/15 split (seed=42)
           └─ Standardizes numeric features (fit on train only)
              └─ Returns: data dict + dataset_info

MODEL INSTANTIATION
  └─ create_model() looks up model class from registry
     └─ build_model_params() calculates parameters from config
        └─ Instantiates model (e.g., MPO2(**params))
           └─ Creates tensor network with quimb
              └─ Extracts: model.tn, model.input_labels, model.output_dims

INPUT PREPARATION
  └─ For NTN: create_inputs() creates Inputs object with batches
     └─ Each batch: (mu=[qt.Tensor, ...], y=qt.Tensor)
  └─ For GTN: create PyTorch DataLoaders
     └─ Each batch: (X_batch, y_batch) as torch.Tensor

TRAINING
  └─ If NTN: run_ntn() uses Newton-based optimization
     └─ Ridge schedule: [5.0, 1.25, 0.3125, ...]
        └─ For each epoch: compute loss, gradient, Hessian
           └─ Update: θ = θ - (hess + ridge*I)^{-1} @ grad
  └─ If GTN: run_gtn() uses gradient descent
     └─ For each epoch: forward, backward, optimizer.step()
        └─ Early stopping based on validation quality

RESULT SAVING
  └─ Save results.json with metrics, config, dataset_info
     └─ Update tracking CSV (if enabled)

================================================================================
                        IMPORTANT NOTES
================================================================================

1. SMALL REGRESSION DATASETS
   - There are NO small regression datasets in the current collection
   - Smallest regression dataset is "concrete" with 165 samples (Medium)
   - For small datasets, use classification datasets (iris, wine, breast, etc.)

2. DATASET SPLITS
   - Train/val/test splits are FIXED (seed=42) for reproducibility
   - Experiment seed (default 42) controls model initialization, NOT data splits
   - This is standard practice to isolate model variance from data variance

3. BIAS TERM
   - Standard models: bias term appended to features (input_dim = raw_features + 1)
   - TNML models: NO bias term (uses feature encoding instead)

4. FEATURE ENCODING
   - Standard models: raw features used directly
   - TNML_P: polynomial encoding [1, x, x^2, ..., x^degree]
   - TNML_F: Fourier encoding [cos(x*π/2), sin(x*π/2)]

5. RIDGE REGULARIZATION
   - NTN: ridge parameter decays per epoch (ridge_decay=0.25)
   - GTN: weight_decay = 2 * ridge (to match NTN behavior)

6. MODEL SELECTION
   - Default: select model with best validation quality
   - With train_selection=true: use training quality as tiebreaker when val quality equal
